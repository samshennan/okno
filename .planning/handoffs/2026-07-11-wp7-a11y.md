# Handoff — WP7: a11y pass on icon buttons (2026-07-11)

## What
- The app's ONE icon-only button (settings hidden-grid restore button,
  dynamically created) now gets `aria-label="Restore photo"` (lucide
  stamps `aria-hidden` on all icon SVGs, so the button had only a `title`).
- Six nameless settings toggle checkboxes got static aria-labels matching
  their visible `.name` text (same nameless-control defect class).
- `aria-pressed` added to the class-toggled groups whose state was
  visual-only: slideshow interval + display-mode controls, settings
  interval/display-mode/transition-time groups — set in markup as "false"
  and flipped in lockstep wherever `.active` toggles.
- Deliberately SKIPPED `aria-pressed` on #playPauseBtn/#fullscreenBtn:
  their labels swap (Play/Pause, Fullscreen/Exit Fullscreen); changing
  label + aria-pressed together is an ARIA anti-pattern. Attributes on
  those buttons would survive the innerHTML rewrites, so this was a
  design choice, not a technical constraint.
- Keyboard help: "Esc Exit" → "Esc Exit (fullscreen first)".
- New test asserts: restore aria-label, toggle aria-labels, non-empty
  text names on the 7 slideshow controls, aria-pressed flips on tap
  (previous-active derived at runtime — the shared stub config is mutated
  by the settings-save test), keyboard help mentions fullscreen.
